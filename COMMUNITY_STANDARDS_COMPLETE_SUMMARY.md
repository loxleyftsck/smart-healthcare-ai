- User security checklist pre-deployment

1. Email security team privately
2. 24-hour acknowledgment
3. Patch within 3 days
4. Advisory published

## 🚀 Deployment Instructions

### For GitHub Setup

1. **Create Repository on GitHub**
2. **Upload all files:**
   ```bash
   git add .
   git commit -m "docs: add community standards"
   git push origin main
   ```

3. **Enable in GitHub Settings:**
   - Settings → Code and automation → Rules
   - Add branch protection: require PR reviews
   - Add branch protection: require status checks

4. **Configure GitHub Features:**
   - Enable Discussions
   - Enable Project board
   - Enable security advisories
   - Add code owners (.github/CODEOWNERS)

5. **Add to Repository About Section:**
   - Description: "AI-powered healthcare system"
   - Website: GitHub project link
   - Topics: healthcare, ai, laravel, python
   - Code of Conduct: Select from template

### For Local Development

1. **Install pre-commit hooks**
   ```bash
   # Automatically run standards checks
   git config core.hooksPath .githooks
   ```

2. **Run linters before commit**
   ```bash
   # PHP
   ./vendor/bin/pint
   
   # Python
   black ai-triage-service/
   ```
